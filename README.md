### Практика в городской администрации 
Интеграция модуля уведомлений граждан.


### 1. Анализ «документации» 
## ТРЕБОВАНИЯ К МОДУЛЮ УВЕДОМЛЕНИЙ ГРАЖДАН
1. Общая информация
   Система: "Электронная приемная администрации"
   Назначение: автоматическое оповещение граждан об изменении статуса обращений

2. Функциональные требования
   2.1. При изменении статуса обращения модуль должен отправлять уведомление
   2.2. Способ отправки: SMS (при наличии телефона) или Email (при наличии email)
   2.3. Текст уведомления: "Ваше обращение №{ID} переведено в статус: {СТАТУС}"
   2.4. Если нет контактных данных - система логирует предупреждение

3. Структура данных (обращение гражданина)
   - id: уникальный номер
   - text: текст обращения
   - status: текущий статус
   - phone: номер телефона (может быть пустым)
   - email: электронная почта (может быть пустой)

4. Статусы обращений
   - Принято
   - На рассмотрении
   - В работе
   - Исполнено
   - Отказ
